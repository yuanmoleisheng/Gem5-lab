## step3:分析运行结果

一、影响程序⾏为的属性有 a) Memory Regularity b) Control Regularity c) Memory Intensity d)Memory Locality等。请谈谈你对这些属性的理解，并且简要分析你选择的基准测试程序在这些属性上的差异。

1. Memory Regularity（内存访问规律性）

指内存访问地址的可预测性，核心是访存模式是否能提前预判。

* 高规律性：访问地址按固定模式排列（如顺序递增、固定步长跳步），CPU / 缓存可通过预取优化；
* 低规律性：访问地址无固定模式（如随机生成索引），预取失效，缓存命中率易降低。

2. Control Regularity（控制流规律性）

指程序执行路径的可预测性，核心是分支跳转是否可预判。

* 高规律性：程序以循环、顺序执行为主，分支少（如单循环无条件判断），分支预测命中率接近 100%；
* 低规律性：程序含大量条件分支（如 if-else 嵌套、switch 多分支）或动态跳转，分支预测易失效，导致 CPU 流水线阻塞。

3. Memory Intensity（访存强度）

指程序执行过程中访存操作与算术运算的比例，核心是“程序对内存的依赖程度”。

* 高访存强度：每执行 1 条算术指令需伴随多次内存读写（如纯数组遍历）；
* 低访存强度：算术运算密集，内存读写极少（如纯计算），性能瓶颈在 CPU 运算单元。

4. Memory Locality（内存局部性）

指程序对内存地址的重复访问特性，分为时间局部性和空间局部性。

* 高局部性：频繁访问同一或相邻地址（如循环遍历数组），缓存易命中；
* 低局部性：访问地址分散，无重复或邻近访问（如随机访问大数组），缓存易失效。

| Memory Regularity |       高       |        低        |     中     |
| Control Regularity |       高       |        高        |     高     |
|  Memory Intensity  |       低       |        高        |     中     |
|  Memory Locality  |       中       |        低        |     高     |

二、从测试结果的 stats.txt 中，列举⼀个/⼀组指标，反应上述属性的差异。例如，是否可以通过某个指标的⼤⼩判断程序的Memory Locality程度？

从 `pageHitRate`这一个指标中可以看出程序的Memory Locality程度，因为如果一个程序的局部性较好，那么其访存的命中率应该会比较高。从三个基准测试程序中我们能看到数组随机访问的命中率最低，因为它访问的数组元素在内存中可能隔得很远，导致频繁Miss；命中率最高的是混合负载程序，因为它的内存访问模式几乎都是顺序的；位于二者之间的矩阵乘法由于其右矩阵的计算模式是列主序，但内存是行主序，在未优化时它的命中率不会太高。
